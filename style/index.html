# 基本约定

  1. 使用正确的类名。错误示例：.a1、.biaoti、.wzdb

  2. 不单独使用重复率高的类名。示例：.title、.btn、.button

  3. 不使用 ID 选择器。

    1. 页面中可以使用 ID，仅仅用作<label>、<input>、<a>配合使用
    2. 允许 CSS 中有且仅有一个 ID 选择器。

  4. CSS 选择器请勿嵌套多层。

  5. 请勿在 CSS 内嵌套使用标签选择器。

    why？

    如果我们写下这样一行代码：

    .article-section h2 {
  color: #f00;
}

    过一段时间后，将<h2>替换成了<h3>那么原先设置的样式便不再生效，而又需要修改 CSS 代码。听上去是挺麻烦的一件事，如果修改的标签数量较多，那么 CSS 代码也需要修改。

    所以，不要在 CSS 内嵌套标签使用。（也配合排版引擎解析规则）

# 书写顺序

我们定义好 CSS 的书写顺序，以减少代码混乱无序、重复出现的可能。

  1. 位置属性（position、top、right、z-index、display、float等）
  2. 大小（padding、margin、width、height）
  3. 文字系列（font、line-height、letter-spacing、color、text-align等）
  4. 背景（background、border等）
  5. 其他（animation、transition等）
/* 不应该这样做 */
.nav {
  top: 0;
  background: #ffffff; /* 应当简写 */
  width: 100%;
  padding: 10px;
  left: 0;
  font-size: #666666;
  position: fixed;
  height: 50px;
}

/* 应该这样做 */
.nav {
  position: fixed;
  top: 0;
  left: 0;
  padding: 10px;
  width: 100%;
  height: 50px;
  font-size: #666;
  background: #fff;
}

/* 栗子 */
.example {
  position: relative;
  top: 0;
  right: 0;
  bottom: 0;
  left: 0;
  z-index: 1;
  display: inline-block;
  width: 300px;
  font-size: 24px;
  text-align: center;
  color: #f1f1f1;
  border: 1px solid rgba(98, 98, 98, .8);
  background-color: #ddd;
  transform: rotate(45deg);
  tramsition: transform .3s;
}

# 前缀

在类名中，我们会加一点前缀，以表当前的类名归类或状态。我们定义以下几类：

module（模块）：.m-表示一个可重复使用的较大的整体，例如：导航、列表、表单……

unit（元件）：.u-通常表示不能够再次细分的小巧个体，例如：Button、input、loading……

skin（皮肤）：.s-单独抽离的样式，例如：阴影、背景色、字体颜色……

function（功能）：.f-应当应用于 JavaScript DOM 操作中，不应出现在样式表内。

注意：以上前缀即表示全局作用，增加与调整应当充分考虑。

# 项目约定

在开始编写业务代码前，我们需要做如下操作：

  1. 重置标签默认样式。（reset.css）

  2. 定义全局复用样式。（global.css）

在global.css中编写所有相同的复用样式，例如每个内容区域的标题、副标题等样式。发现可重复使用的样式应当立即写入全局样式表，并写好文档，告知其他成员使用。

注意：像按钮、文本框等此类不应该写入全局样式表，应该写入公共组件样式表（module.css）。

# reset

尽可能不使用通配符：* 清除内外边距或设置其他样式。

reset.css应当仅作为样式重置使用，不要在样式重置时为标签添加自定义样式。

例如：

/* 错误行为 */
button {
  background: blue;
}

# Block

首先，块是可嵌套的。它们应该能被包含在另一个块中，而不会破坏任何样式。例如，在侧栏中有一个标签界面小部件的块，该块可能包含按钮，这些按钮也是一种单独的块。按钮的样式和选项卡式元素的样式不会相互影响，一个嵌套在另一个中，仅此而已。

# Element

元素是块的组成部分，它不能在块之外使用。例子：一个导航菜单，它包含的项目在菜单的上下文之外没有意义。你不会为菜单项定义块，菜单本身应定义为块，而菜单项是其子元素。

# 修饰符（Modifier）

修饰符定义块的外观和行为。例如：按钮的颜色，取决于所使用的修饰符。

# 命名约定

BEM 定义了非常严格的命名约定，但是我们修改一下：

<button class="u-button">Base Button</button>
<button class="u-button u-button-state--success">Success Button</button>
<button class="u-button u-button-state--danger">Danger Button</button>

.u-button {
  display: inline-block;
  padding: 10px;
  font-size: 16px;
  border: none;
  border-radius: 4px;
}
.u-button--success {
  color: #fff;
  background: #00d055;
}
.u-button--danger {
  color: #fff;
  background: green;
}
